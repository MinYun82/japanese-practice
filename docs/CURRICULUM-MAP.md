# 全書課程地圖 — 大家的日本語 初級Ⅰ 自學版

> 建立：2026-07-12。目標：在沒有老師上課的情況下，把《大家的日本語 初級Ⅰ》（L1–L25）
> 全書切成等分量的 7 天自學單元，媽媽每天打開網頁就有當天的練習。

## 設計原則

- **一單元 = 一週 = 7 天**，沿用既有節奏：認識新字 → 對比 → 小測驗 → 練習 → 綜合複習 → 總測驗
- **每單元約 20–30 個項目**（假名單元可到 40，因為假名認知負荷低於單字句型）
- **每課教科書內容拆 1–2 個單元**，文法多的課（如て形）拆兩個
- **概念卡（concepts）承擔老師的角色**：每個新文法都先有白話解說＋例句，才進翻卡
- **句型卡用完整例句**（畫面＝聲音），套用媽媽的名字みさお
- 單元 `date` 依序每週一開始，預排未來；首頁自動選「今天所在的週」
- 媽媽若某週沒跟上：週次選單可手動切回上一週，進度不會消失（localStorage）

## 產出新單元時的檢查清單

1. `date` = 該單元開始的週一（接在前一單元 +7 天）
2. 所有項目放 `newItems` 用 `type` 區分（vocab / grammar / yoon / katakana…），不開新陣列
3. 每個文法概念先寫 `concepts` 卡（媽媽看得懂的白話＋例句）
4. 句型卡：`char` 用完整例句（假名＋空格），`furigana` 標漢字讀音，人名用みさお
5. `chineseHint` 給發音諧音＋意思；`zhuyinHint` 給注音
6. `compareWith` 只能指向**同一週**的 newItems（跨週配不到會被靜默跳過）
7. dailyPlan 7 天、mode 只用 `concepts` / `flipcard` / `compare` / `quiz`（`+` 串接）
8. 複習（`review`）帶前幾週的舊 id（findById 會跨週搜尋 ALL_WEEKS）
9. id 全域唯一——先跑驗證腳本再交付（見 PITFALLS #21 條目附近說明）
10. 改完 `date +%s > version.txt`，跟 all-weeks.js 一起 commit + push
